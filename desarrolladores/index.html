Para Desarrolladores

Contribuye a Fiscalius, crea extensiones o integra el sistema con otras aplicaciones.

Únete a la Comunidad

Fiscalius es un proyecto open source que necesita tu ayuda para crecer. Puedes contribuir de muchas maneras:

  • Reportar bugs - Encuentra y documenta errores
  • Mejorar documentación - Ayuda a otros a entender el sistema
  • Crear extensiones - Desarrolla módulos adicionales
  • Integrar sistemas - Conecta Fiscalius con otras aplicaciones
  • Mejorar el código - Optimiza y refactoriza existente
  • Diseñar interfaces - Mejora la experiencia de usuario

Arquitectura del Sistema

Fiscalius utiliza una arquitectura cliente-servidor de 3 capas:

┌─────────────────────────┐   hb_netio TCP/IP   ┌───────────────────────┐   SQL   ┌────────────┐
│  Fiscalius Client       │ ←─────────────────→  │ Fiscalius Server      │ ←─────→ │ PostgreSQL │
│  (TPuy/xHarbour + GTK)  │                       │ (Harbour/TPuy)        │         │ 16         │
│  UI + Validaciones prev.│                       │ API + Lógica negocio  │         └────────────┘
└─────────────────────────┘                       └───────────────────────┘

Componentes Principales

  • Cliente: Interfaz gráfica GTK+ desarrollada en TPuy (xHarbour + T-GTK)
  • Servidor: API RESTful desarrollada en Harbour/TPuy
  • Base de datos: PostgreSQL 16
  • Comunicación: Protocolo NetIO (hb_netio)

Tecnologías Utilizadas

🔧

TPuy Framework

Framework propio basado en Harbour + T-GTK para desarrollo de aplicaciones de escritorio con interfaces GTK+.

Harbour

Compilador de alto nivel con capa de desarrollo xBase. Permite elaborar interfaces con un lenguaje de rápido aprendizaje, robusto y probado.

PostgreSQL

Sistema de base de datos relacional robusto, escalable y de código abierto.

🌐

NetIO

Protocolo de comunicación cliente-servidor basado en TCP/IP para transfers seguras.

Guías de Desarrollo

Primeros Pasos

  1. Clona el repositorio: 
    git clone https://github.com/riztan/fiscalius-client.git
cd fiscalius-client
  2. Instala las dependencias: 
    # En Linux
sudo apt-get install harbour tgtk

# En Windows
# Descarga TPuy desde https://github.com/riztan/tpuy
  3. Compila el proyecto: 
    tpuy
  4. Ejecuta: 
    ./fiscalius-client

Patrones de Código

Al desarrollar para Fiscalius, sigue estos patrones:

Llamada a servidor (NetIO):

// Verificar conexión primero
IF !oTPuy:RunXBS("netio_check")
   MsgStop("Sin conexión al servidor")
   RETURN
ENDIF

// Obtener objeto remoto
::rFinanzas := oFormParent:oGEmpresa:SetModulo("Finanzas")
::rCobranza := ::rFinanzas:Cobranza('')

// Verificar respuesta
IF hb_isObject( ::rCobranza )
   hData := ::rCobranza:Data()
ENDIF

Cargar datos a ListBox:

oDatos := rEmpresa:Metodo( hParams )
oModel:oTreeView:ClearModel(.T.)
FOR EACH aLine IN oDatos:GetData()
   APPEND LIST_STORE oModel:oLbx ITER aIter VALUES aLine[1], aLine[2], aLine[3]
NEXT

Reglas Importantes

  • NO usar VAL() - Usar siempre ToNum() para números con decimales
  • Verificar objetos - Usar hb_isObject() antes de acceder
  • LOCAL al inicio - Declarar variables locales al inicio del procedimiento
  • No modificar archivos funcionales - Preguntar antes de cambiar algo que funciona
