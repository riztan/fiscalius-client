Preguntas y Respuestas Técnicas

Respuestas a las preguntas técnicas más frecuentes sobre Fiscalius.

1. Arquitectura y Diseño

¿Por qué se usa una arquitectura cliente-servidor?

La arquitectura cliente-servidor ofrece varias ventajas:

  • Seguridad: Los datos se almacenan centralmente
  • Concurrencia: Múltiples usuarios pueden trabajar simultáneamente
  • Mantenimiento: Actualizaciones centralizadas
  • Escalabilidad: Se pueden agregar más clientes sin cambios

¿Por qué se usa Harbour en lugar de otros lenguajes?

Harbour es un lenguaje robusto y probado que:

  • Tiene excelente soporte para bases de datos
  • Es compatible con Clipper (legado de sistemas anteriores)
  • Tiene buena integración con GTK+ a través de T-GTK
  • Es de código abierto y gratuito

¿Por qué no se usa una arquitectura web?

Aunque una arquitectura web tendría ventajas, se eligió cliente-servidor porque:

  • Mejor rendimiento en operaciones de facturación
  • Funciona sin conexión a Internet (una vez autenticado)
  • Mejor integración con hardware (impresoras, lectores)
  • Experiencia de usuario más fluida

2. Comunicación Cliente-Servidor

¿Cómo funciona la comunicación NetIO?

NetIO es un protocolo de comunicación basado en TCP/IP que:

  • Utiliza el puerto 2942 por defecto
  • Transmite datos en formato JSON
  • Es seguro y confiable
  • Permite comunicación bidireccional

¿Qué objetos se comunican con el servidor?

Los objetos principales son:

  • oGEmpresa: Objeto principal de conexión
  • rVentas: Módulo de ventas
  • rFinanzas: Módulo de finanzas
  • rInventario: Módulo de inventario

¿Cómo se manejan los errores de conexión?

IF !oTPuy:RunXBS("netio_check")
   MsgStop("Sin conexión al servidor")
   RETURN
ENDIF

3. Base de Datos

¿Por qué se usa PostgreSQL?

PostgreSQL ofrece:

  • Alta performance y confiabilidad
  • Soporte para transacciones ACID
  • Escalabilidad y flexibilidad
  • Es de código abierto y gratuito
  • Excelente soporte para consultas complejas

¿Cómo se conecta el cliente a la base de datos?

El cliente NO se conecta directamente a la base de datos. La comunicación es:

Cliente → Servidor → PostgreSQL

El servidor maneja todas las operaciones de base de datos.

¿Cómo se manejan las transacciones?

Las transacciones se manejan en el servidor utilizando el motor de PostgreSQL. El cliente envía operaciones y el servidor las ejecuta de manera transaccional.

4. Interfaz de Usuario

¿Cómo se crean las interfaces?

Las interfaces se crean con:

  • Glade: Diseñador visual de interfaces GTK+
  • Archivos .ui: Definición XML de las interfaces
  • T-GTK: Binding de Harbour para GTK+

¿Cómo se cargan los datos en los ListBox?

oModel:oTreeView:ClearModel(.T.)
FOR EACH aLine IN oDatos:GetData()
   APPEND LIST_STORE oModel:oLbx ITER aIter VALUES aLine[1], aLine[2], aLine[3]
NEXT

¿Cómo se manejan los eventos?

Los eventos se manejan con:

  • CONNECT: Conectar eventos a funciones
  • ON CLICK: Eventos de clic
  • ON CHANGE: Eventos de cambio

5. Seguridad

¿Cómo se autentican los usuarios?

La autenticación se realiza en el servidor:

  1. El cliente envía usuario y contraseña
  2. El servidor verifica las credenciales
  3. Si son correctas, devuelve un token de sesión
  4. El cliente almacena el token para futuras peticiones

¿Cómo se protegen los datos sensibles?

  • Las contraseñas se almacenan hasheadas
  • La comunicación puede usar SSL/TLS
  • Los datos sensibles se almacenan en el servidor
  • Se realizan auditorías de acceso

¿Cómo se manejan los permisos?

Los permisos se controlan por:

  • Roles: Admin, Contador, Vendedor, etc.
  • Módulos: Acceso a módulos específicos
  • Acciones: Crear, editar, eliminar, imprimir

6. Rendimiento

¿Cómo se optimiza el rendimiento?

  • Consultas optimizadas: Índices en PostgreSQL
  • Caching: Datos estáticos en caché del cliente
  • Compresión: Datos comprimidos en red
  • Conexión persistente: Evita overhead de conexión

¿Cómo se maneja el volumen de datos?

  • Paginación de resultados
  • Consultas con filtros
  • Exportación a archivos
  • Archivado de datos antiguos

7. Desarrollo

¿Cómo agregar una nueva función?

  1. Crear el archivo .xbs en xbscripts/
  2. Definir la interfaz en resources/
  3. Conectar la interfaz con la lógica
  4. Probar la función
  5. Documentar la función

¿Cómo modificar una función existente?

  1. Entender la función actual
  2. Identificar los puntos de modificación
  3. Realizar los cambios
  4. Probar exhaustivamente
  5. Documentar los cambios

¿Cómo depurar el código?

  • MsgStop: Mostrar mensajes de depuración
  • Log: Escribir en archivo de log
  • Console: Salida a consola
  • Debugger: Usar debugger de Harbour

8. Despliegue

¿Cómo se distribuye el cliente?

El cliente se distribuye como:

  • Windows: Instalador .exe
  • Linux: Paquete .deb o .rpm
  • Portable: Archivo comprimido

¿Cómo se actualiza el cliente?

Las actualizaciones se realizan:

  • Automáticamente: Descarga de actualizaciones
  • Manualmente: Descarga desde GitHub
  • Por distribuidor: Paquetes de actualización

9. Solución de Problemas

¿Qué hacer si el cliente no se conecta al servidor?

  1. Verificar que el servidor esté activo
  2. Verificar la configuración de red
  3. Verificar firewall
  4. Verificar credenciales
  5. Revisar logs del servidor

¿Qué hacer si hay errores de base de datos?

  1. Verificar conexión a PostgreSQL
  2. Verificar permisos de usuario de BD
  3. Verificar espacio en disco
  4. Revisar logs de PostgreSQL

¿Qué hacer si la interfaz no responde?

  1. Reiniciar la aplicación
  2. Verificar memoria disponible
  3. Verificar procesos bloqueados
  4. Revisar logs de la aplicación
